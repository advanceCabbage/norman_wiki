## 一、名词解释

- **CodeBase**： 指一个软件项目的所有源代码文件，包括代码、配置文件、脚本、资源文件等，构成了项目的整体代码基础，代码仓库是一种存储 CodeBase 的地方。

- **LSP**：**Language Server Protocol (LSP)** 是由 Microsoft 2016年提出并维护的一种开放协议，**用于在代码编辑器或 IDE 与语言服务器之间建立标准化通信**。它的目标是，让编辑器能够以统一的方式支持各种编程语言的智能功能（如自动补全、跳转定义、错误提示等），而无需为每种语言和每个编辑器重复实现。

- **Serena**：Serena 是一个开源编程代理工具包，**可将大语言模型（LLM）转化为具备 IDE 级功能的开发代理，支持在代码库上进行语义检索与编辑**。基于语言服务器协议（LSP），它涵盖 Python、JavaScript、Java、Rust 等 15+ 种语言，提供符号级代码分析，显著提升效率与理解精度。不依赖特定 LLM 或框架，既可通过模型上下文协议（MCP）无缝集成 CatPaw等客户端，也可自由集成到[Agno](https://docs.agno.com/introduction)等代理框架中。被誉为“游戏规则改变者”，大幅增强 LLM 在复杂代码场景下的开发效能。

## 二、背景

当前MSI官网文档存在部分API文档与四端源码实现不一致问题。MSI涵盖API数量众多，假设由人工去校准API与四端源码差异，将会有产生巨大的工作量耗时。基于以上背景，期望借助AI + CodeBase能力实现智能分析四端源码、与官网文档进行差异对比、自动生成准确的文档草稿全自动流程。

## 三、目标

1. **API文档准确性校准**：通过仅输入API名称，识别API所属代码仓，LLM调用语言服务器分析四端仓库源码，依据分析结果与官网API文档进行差异对比，生成文档差异报告。借助AI能力快速且准确的识别API文档问题，帮助完善官网文档。

2. **API文档草稿智能生成**：用户仅需指定API名称和代码仓库分支（不传则走主分支逻辑），LLM调用语言服务器分析四端仓库源码，整理融合四端API文档分析结果，调用生成文档草稿open api。既能生成本次需求迭代API文档也能检测出历史文档问题，保证API文档准确性。

在整体架构设计上，我们采用功能职责单一性原则，将服务能力划分为两个独立模块：MSI Doc Agent Server和CodeBase Server。这种设计具有显著优势，实现了业务逻辑与代码检索能力的彻底解耦。其中，MSI Doc Agent Server同时支持MCP和HTTP两种协议，而CodeBase Server则专注于通过HTTP接口提供代码相关服务。

MSI Doc Agent Server主要负责以下核心业务逻辑：

1. API代码仓定位：通过解析接收到的API名称，智能识别对应的实现组件。基于预设的组件与代码仓库映射关系，精准定位该API在Android、iOS和HarmonyOS三大平台的代码仓库位置。

2. Prompt智能组装：为Android、iOS、HarmonyOS以及MSI-FE四个平台的任务动态构建Prompt。每个Prompt都包含关键信息：需要激活的项目名称、详细的任务分析要求、特定任务的约束条件以及标准化的输出格式要求。

3. 多任务并发执行：采用并行处理机制同时执行四端分析任务，并支持流式记录分析过程。系统会实时更新并展示各端任务的进度状态，确保过程透明可控。

4. 智能文档生成：整合四端分析结果后，系统会自动获取官网API文档详情，并利用大语言模型（LLM）进行智能比对，最终生成专业的源码实现与官方文档差异报告。

CodeBase Server则专注于代码层面的核心功能：
该服务基于开源的Serena框架深度定制优化，专门处理代码仓库的检索与编辑任务。Serena是一个功能强大的开源编程代理工具包，能够将大语言模型转化为具备完整IDE级功能的智能开发代理。其核心优势包括：

- 支持15+种主流编程语言（包括Python、JavaScript、Java、Rust等）

- **基于语言服务器协议（LSP）实现精准的符号级代码分析**

- 提供高效的语义检索和智能编辑能力

- 框架中立设计，不绑定特定LLM或开发框架

- 显著提升代码理解精度和开发效率

在后续章节中，我们将深入解析Serena框架的实现原理、LSP协议的工作机制以及Agno技术的核心思想。

## 六、Serena实现CodeBase方案原理

### 6.1 Language Server Protocol

#### 6.1.1 引言：为什么需要LSP？

在当今软件开发领域，开发者们高度依赖编辑器或集成开发环境（IDE）提供的智能功能来提升编码效率。这些功能包括代码自动补全、定义跳转、悬停提示以及实时错误诊断等。然而，在语言服务器协议（LSP）问世之前，这些功能的实现面临着严重的资源浪费和生态碎片化问题。

传统开发模式存在三大核心痛点：

**首先，编辑器之间重复建设严重**。以Python语言支持为例，VS Code、Vim、Emacs和Sublime Text等主流编辑器都需要独立开发各自的语法解析和代码补全系统，造成了巨大的开发资源浪费。

**其次，语言维护者负担过重**。像Go、Rust这样的编程语言想要获得主流编辑器的良好支持，其维护团队不得不为每个编辑器单独开发适配插件，极大地增加了维护成本。

**最后，用户体验参差不齐**。由于不同编辑器对同一语言的支持程度存在差异，开发者在切换开发工具时往往需要重新适应，这不仅增加了学习成本，还显著降低了工作效率。

2016年，微软推出的语言服务器协议（LSP）为这些问题提供了革命性的解决方案。该协议的核心创新在于将语言智能功能与编辑器界面进行解耦：

**LSP创造性地将语言分析能力从编辑器中剥离出来，封装成独立运行的语言服务进程。通过标准化的通信协议，这个服务可以与任何支持LSP的编辑器进行交互**。这种架构带来了两大突破性优势：

一方面，编辑器开发者只需实现一次LSP客户端，就能无缝接入所有支持该协议的语言服务；另一方面，语言维护者仅需开发一个语言服务器，就能让所有兼容LSP的编辑器获得完整的智能功能支持。

这种创新架构真正实现了"一次开发，全平台通用"的语言智能服务生态，彻底改变了软件开发工具的发展格局。

#### 6.1.2 LSP核心架构与通信模型

语言服务器协议（LSP）采用客户端-服务器架构与JSON-RPC协议，构建了一套松耦合、跨平台且高度可扩展的语言服务解决方案。其核心架构由两大组件构成：

- **Client（客户端）：**在客户端方面，编辑器集成组件（如VS Code、Neovim等）负责处理用户交互界面展示，并向服务器发起各类语言服务请求（例如代码补全建议获取）。

- **Server（语言服务器）：**服务端则由独立运行的语言服务器进程（如gopls（Go）、pyright（Python）等）构成，专门处理请求并返回结构化语言数据（如补全项列表、类型信息）。

通信机制基于标准化的JSON-RPC 2.0协议，所有消息均采用JSON格式，主要包含三种交互模式：

- 请求/响应模式（带唯一ID标识）

- 单向通知模式（无需响应）

- 错误响应机制（包含错误代码和描述信息）

以代码补全为例，典型交互流程如下：

```JSON
// 客户端发起请求 - 语言服务器响应示例
// Client 发送请求
{

  "jsonrpc": "2.0",

  "id": 1,

  "method": "textDocument/completion",

  "params": {

    "textDocument": { "uri": "file:///example.py" },

    "position": { "line": 5, "character": 10 }

  }

}

// Server 返回响应

{

  "jsonrpc": "2.0",

  "id": 1,

  "result": [

    { "label": "print", "kind": 3 },

    { "label": "len", "kind": 3 }

  ]

}
```

为了确保客户端和服务端功能兼容，协议初始化阶段采用能力协商机制确保兼容性：

- 客户端通过initialize请求声明支持的功能特性

- 服务端返回InitializeResult响应说明自身能力范围

- 后续交互仅使用双方共同支持的功能集

这种设计使LSP协议具备出色的版本兼容性和功能扩展能力，支持不同版本的客户端和服务端实现无缝协作。

#### 6.1.3  核心语言功能概览

Serena 的语义代码分析能力基于广泛使用的语言服务器协议（LSP）构建。LSP 提供了一套基于代码符号理解的通用代码查询和编辑功能。凭借这些能力，Serena 能够像经验丰富的开发者使用 IDE 功能一样发现和编辑代码。即使在非常庞大和复杂的项目中，Serena 也能高效地找到正确的上下文并采取正确的行动！

语言服务器支持多种编程语言。通过 Serena 的 LSP 库，支持超过 30 种编程语言，包括 AL、Bash、C#、C/C++、Clojure、Dart、Elixir、Elm、Erlang、Fortran、Go、Haskell、Java、Javascript、Julia、Kotlin、Lua、Markdown、Nix、Perl、PHP、Python、R、Ruby、Rust、Scala、Swift、TypeScript 和 Zig。
