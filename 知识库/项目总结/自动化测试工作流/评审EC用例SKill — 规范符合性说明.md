# EC 用例评审 Skill：规范符合性说明

本说明评审 [`评审EC用例SKill.md`](评审EC用例SKill.md) 是否符合“写好 Skill”的两份学习文档：[`SKILL.zh-CN.md`](/Users/haoyang/Documents/AIProject/skills/skills/productivity/writing-great-skills/SKILL.zh-CN.md) 与 [`GLOSSARY.zh-CN.md`](/Users/haoyang/Documents/AIProject/skills/skills/productivity/writing-great-skills/GLOSSARY.zh-CN.md)。

## 结论

重写后的 Skill 采用模型调用型入口、步骤优先的信息层级和按需参考资料。主文件负责稳定的评审过程；EC 细则、评分与模板集中在一个外置参考文件中，避免规则散落或重复。

## 逐项符合性评审

| 规范 | 本 Skill 的实现 | 符合原因 |
| --- | --- | --- |
| 模型调用型 | Frontmatter 保留 `description`，没有 `disable-model-invocation`。 | 模型及其他 Skill 都可发现并调用它，适合协议消息自动触发。 |
| description 的主导词 | description 以“**证据链**”开头。 | “证据链”在正文中反复作为评审行为锚点，连接触发与执行。 |
| description 仅保留触发条件 | description 只说明 EC JSON 评审与 `[评审类型] ec-test-case-spec` 两个触发分支。 | 不把角色、规则、输出格式等正文信息塞进始终加载的 description，降低上下文负担。 |
| 步骤为顶层信息 | 主文件的五步流程是：确认范围、建立证据链、执行规则、生成报告、提交前自检。 | 智能体最先需要的是行动顺序，因而步骤位于 `SKILL.md` 主层。 |

| 充分前置功 | 第 2 步要求读取并解析 JSON、提取 API 文档事实，并为结论保存路径、JSON 路径和片段。 | 将“评审”具体化为可追溯的证据工作，而不是仅给主观结论。 |
| 渐进披露 | EC 1–EC 13、评分和报告模板放在 [`评审EC用例 — 规则与报告模板.md`](评审EC用例%20—%20规则与报告模板.md)。 | 它们只在评审阶段需要，外置后保持主流程短而可读。 |
| 上下文指针清晰 | 主文件的“必要参考资料”说明在何时必须读取哪个文件。 | 指针不仅给出链接，还明确“执行规则、计算评分或生成报告时必须读取”，提高触达可靠性。 |

| 正向引导 | 主流程用“报告只输出所选模板字段”“每个 fail 关联证据”等目标动作表达约束。 | 以目标行为为主；少量“仅”用于协议范围和模板边界这类必要硬约束。 |
| 精简与无效指令控制 | 移除了原文件中重复出现的角色、禁止清单、模板说明和规则细节。 | 重复含义被合并为一次定义；主文件不再承载只在局部使用的长表。 |
