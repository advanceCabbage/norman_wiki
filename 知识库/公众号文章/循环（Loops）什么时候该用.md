## 核心观点

大多数人天天用 AI，却用的是最慢的方式：输入请求 → 等待 → 手动修改 → 再问，全靠手动循环。而"循环（loop）"是把这个过程自动化——让 agent 反复迭代直到产出可用结果。

## 什么时候才值得搭一个循环

只有当下面 4 个条件**同时满足**时，才值得为一个任务搭建循环：

1. **任务会重复**——至少每周一次。频率再低，搭建成本就收不回来；一次性任务用一个好 prompt 就够了。
2. **能自动否决坏结果**——有测试、类型检查、构建、linter 或硬性规则能自动判定失败。否则循环只会空转。
3. **agent 能端到端独立完成**——而不是做一半又甩回给你。
4. **"完成"是客观的**——不是靠主观品味判断。如果质量是审美问题，人还是更强。

## 最有用的结构技巧：写的和查的分开

循环里最有用的结构技巧，是把"干活的 agent"和"检查的 agent"分开：

- 写代码的模型给自己批改作业时"太仁慈"。
- 换一个 agent（不同指令，有时用更强的模型 / 更高 effort）来审查，能抓住第一个 agent 自我说服放过的问题。
- 可以让 writer 又快又便宜，让 reviewer 又慢又严格——这种分工贡献了大部分的质量。

## 提醒

循环工程（loop engineering）是真实趋势，但大多数人现在还不需要重型版本。
