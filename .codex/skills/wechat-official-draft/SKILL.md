# 微信公众号官方发文工作流

直连微信官方接口，用用户自己的 AppID + AppSecret 完成"润色 → 建草稿 → 预览确认 → 发表"闭环。纯 Python 标准库，无需 pip 安装。

> **本 skill 自成一体**：UI 美化（字体/主色/标题/加粗/引用/代码块/图片/**表格斑马纹**）已全部内置在样式引擎里。**不要**再调用 `wechat-article-formatter` 或任何第三方渲染服务（bm.md、limyai 等）。只用本 skill 即可完成整个工作流。

---

## 严格工作流（必须按顺序执行，不要跳步）

触发：用户提供【文章 md 文件】并要求润色、创建草稿、更新或发表公众号文章；封面图可由用户提供，也可自动生成。

## 封面图标准

1. 优先使用用户提供的封面；用户未提供时，必须调用 `imagegen` 生成封面，不要因缺图中断建稿流程。
2. 以横版 `16:9` 为主要比例；若生成工具不能精确指定尺寸，在 prompt 中明确要求宽幅 16:9，并在使用前检查宽高比接近 1.78。
3. 从文章标题、核心结论和关键对象提炼主视觉，让封面在缩略图尺寸也能识别文章主题。
4. 默认不要在图中生成长标题、伪代码、难以辨认的文字或未经授权的品牌标识；需要文字时，只放用户给出的短标题并要求逐字准确。
5. 生成后将选定图片复制到本次工作目录，再作为 `--cover` 参数传给脚本；向用户报告生成结果和文件路径。

## 内容与排版质量门槛

1. 只在工作副本中润色，保留原文观点、事实和源码；不要修改用户原始文件。
2. 用清晰且平行的标题层级组织正文：主章节使用“一、二、三”，同一分析章节中的小点使用 `2.1`、`2.2` 这类连续编号。
3. 展示外部来源时，正文必须显示完整、可点击的 URL，例如 `[https://github.com/owner/repo](https://github.com/owner/repo)`；不要只显示仓库简称。
4. 展示源码时，保持源码和中文释义的边界清楚；需要释义时优先放在代码块外。代码块必须为每个逻辑源码行显示行号，长行折行后仍与代码文本列对齐，不得落入行号列。
5. 统一术语和标签；同一篇文章不要混用“好处”“优势”等同义标签，默认使用“优势”。
6. 建稿或更新前逐项检查：封面比例与主题、标题层级、完整链接、代码块行号与折行、首个 H1 去除、中英文间距，以及工作副本路径。发现问题先修正，再调用微信接口。

**第 0 步 · 前置检查（首次或报错时）**
- 确认项目根目录有 `.env`（含 `WECHAT_APPID` / `WECHAT_APPSECRET`）。
- 运行 `python3 $S token`，能打印 access_token 才继续；报 `40164` 见文末错误码处理。

**第 1 步 · 润色（只做 UI/格式，禁止改动观点与事实）**
1. 读取本 skill 目录下的 `polish-rules.md`。
2. 把原文**复制成一份工作副本**（放 scratchpad 或临时目录），**绝不修改用户原始文件**。
3. 按 `polish-rules.md` 和上方「内容与排版质量门槛」处理工作副本（如标题编号、完整链接、代码行号与折行、段落拆分、中英文间距等，未开启的规则不加）。
4. 「样式类规则」无需手动做，脚本会自动内联注入。

**第 1.5 步 · 准备封面**
1. 有用户封面时，检查比例和主题相关性后使用。
2. 无用户封面时，按「封面图标准」调用 `imagegen` 自动生成横版主题封面，并复制到工作目录。

**第 2 步 · 建草稿**
```bash
python3 $S add --md <工作副本> --title "<标题>" --author "norman" --cover <封面图>
```
- 成功后 media_id 自动写入项目根目录 `.wechat_state.json`。
- 正文里的本地/外链图片会自动上传微信；首个 H1 自动去除。

**第 3 步 · 请用户预览（关键：停在这里等用户）**
- 明确告诉用户：去 **公众号后台 → 内容管理 → 草稿箱 → 打开该草稿 → 预览 → 填写微信号推送到手机**。
- ⚠️ 认证订阅号无法用 API 预览（`message/mass/preview` 报 48001），预览**只能走后台手动这一步**。
- **必须等用户反馈**，不要自作主张继续。

**第 4 步 · 按反馈迭代（可多轮）**
- 用户提 UI 意见 → 修改工作副本或样式 → 更新同一草稿（默认复用上次 media_id 和封面）：
```bash
python3 $S update --md <工作副本> --title "<标题>"
```
- 更新后回到第 3 步，请用户再预览。

**第 5 步 · 确认后发表（必须用户显式确认）**
- 仅当用户明确说"确认发布/发表"时执行；`--yes` 是强制确认位，缺少会被拦下：
```bash
python3 $S publish --yes
```
- 发表为异步，之后可在后台「发表记录」查看。

## 前提条件

1. **账号权限**：`draft/add`、`draft/update`、`freepublish/submit` 需已认证的服务号/订阅号。
2. **IP 白名单**：公众号后台 → 开发接口管理 → IP白名单，加入调用方公网 IP。否则报 `40164`。
3. **凭证**：项目根目录 `.env`（参考 `.env.example`）：`WECHAT_APPID` / `WECHAT_APPSECRET`。

## 命令速查

脚本（项目级 skill，从项目根目录运行）：`.codex/skills/wechat-official-draft/scripts/wechat_draft.py`（下称 `$S`）

| 命令 | 说明 |
|---|---|
| `python3 $S token` | 测凭证 + IP 白名单 |
| `python3 $S add --md <f> --title <t> --cover <img> [--author --digest --source-url --open-comment --no-style]` | 新建草稿（先由用户提供或自动生成封面） |
| `python3 $S update [--media-id <id>] --md <f> --title <t> [--cover <img>]` | 更新草稿（默认取 state 的 id，不传 --cover 复用上次封面） |
| `python3 $S publish [--media-id <id>] --yes` | 正式发表（须 --yes） |
| `python3 $S state` | 查看当前记忆的草稿 |
| `python3 $S preview --media-id <id> --to <微信号>` | 单独推预览（仅服务号可用） |

## 样式引擎（美化，全部内置）

- Markdown→HTML：优先 `markdown` 库，否则内置转换器（标题/加粗/斜体/列表/引用/代码块/链接/图片/**表格**）。
- 内联注入样式（默认开启，`--no-style` 关闭）：绿色主题 `#35b378`、Optima/微软雅黑字体、H2 白字绿底、H3 绿字绿边、加粗变绿、引用绿边浅底、公众号稳定代码块、行内代码红字、图片圆角居中、**表格表头绿底+隔行斑马纹**。
- 微信会剥离 `<style>`/class，故样式逐标签内联；全程本地生成，不依赖任何第三方。

### 公众号代码块规则（重要）

微信公众号保存/预览会过滤代码块中的普通换行符，不能依赖 `\n`、单个 `<code>` 内的多行文本，或 `code-snippet__fix` 结构来保留换行。

脚本生成代码块时必须遵守：

- 每一行代码生成一个独立的块级 `<code style="display:block">...</code>`，并在同一行渲染行号。
- 不要把多行内容塞进同一个 `<code>`，否则预览时会变成 `callvalidateInputcheckPermissions` 这类拼接文本。
- 行首空格必须转为 `&nbsp;`，否则缩进可能被微信过滤。
- 空代码行用 `&nbsp;` 占位，不要用 `<br>` 撑高度。
- 行号与代码行必须成对渲染；长行折行时，续行保持在代码文本列内，不得移到行号列下方。不要用 `<br>` 拼接块级行号，否则代码块底部会出现多余空白。
- 不要恢复旧的黑色 `<pre style="background:#2d2d2d">` 方案；手机预览容易横向溢出且不可控。

## 常见错误码

- `40164`：当前公网 IP 未在白名单 → 后台加入。
- `48001` unauthorized api：账号无该接口权限（订阅号无群发/预览权限属正常）。
- `40007` invalid media_id：草稿 id 失效（多为草稿已被删）→ 重新 add。
- `45004` digest too long：摘要超 120 字节 → 脚本已自动按字节截断，一般不会遇到。
- `40125` invalid appsecret：AppSecret 错误或已重置。
