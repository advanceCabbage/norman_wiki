# 公众号润色规则（可编辑）

> 这是你的润色偏好单一来源。每次发文前，工作流会读取本文件并套用。
> 只做 **UI / 格式** 层面的润色，不改动你的观点和事实。改这里即可，无需改代码。

## A. 内容类规则（发文前由智能体套用到正文）

- [ ] **开头关注引导**：在正文最前面加一句，如："👆 点击上方蓝字「XXX」，关注并设为星标，不错过每篇更新"
  - 开启：`否`　引导文案：`（留空）`
- [ ] **结尾互动 CTA**：结尾加一句互动，如："觉得有用就点个赞和「在看」吧～"
  - 开启：`否`　文案：`（留空）`
- [x] **移除正文首个 H1**：微信用标题字段单独显示，避免重复（脚本已自动处理）
- [x] **作者署名**：用 `--author` 传入，默认 `norman`
- **段落长度**：过长的段落适当拆分，每段不超过 3~4 行
- **中英文/数字间距**：中英文之间留空格（如 "使用 AI 工具"）
- **emoji**：正文克制使用，仅小标题或引导语可少量点缀
- **标题层级**：主章节使用“一、二、三”；同一分析章节的小点使用 `2.1`、`2.2` 等连续编号
- **外部链接**：显示完整、可点击的 URL，不只保留仓库名或域名
- **源码展示**：源码与中文释义分开；每个逻辑行保留行号，长行折行后与代码文本列对齐
- **术语一致性**：同一文章使用一致标签，默认将“好处”统一为“优势”
- **推荐内容排布顺序**（参考，按需套用）：关注引导 → 正文 → 分隔线 → 作者介绍 → 分隔线 → 互动 CTA

## B. 样式类规则（由脚本内联注入，改动需同步告知我）

- **主色**：`#35b378`（绿色）
- **正文**：16px，行距 1.85，字色 #3f3f3f
- **H2 二级标题**：白字 + 主色底 + 圆角
- **H3 三级标题**：主色字 + 主色左边框
- **加粗**：主色
- **引用块**：主色左边框 + 浅底
- **代码块**：使用公众号稳定浅色代码块；每个逻辑行各自渲染行号和块级 `code`，续行与代码文本起始位置对齐；禁止用黑色自定义 `pre` 方案
- **代码块换行**：不要依赖 `\n` 或单个 `code` 内多行文本；行首空格转 `&nbsp;`；空行用 `&nbsp;` 占位；行号不要用 `<br>` 叠加，避免底部多余空白
- **行内代码**：浅灰底 + 红字
- **图片**：最大宽度 100%、圆角、居中
- **表格**：表头主色底白字、单元格描边、隔行浅色斑马纹（自动解析 Markdown 表格）

> 想改配色/字号/间距，直接改上面的值并告诉我"按 polish-rules 更新样式"，我会同步到脚本的样式引擎里。

## C. 发布默认项

- 留言：`关闭`（需要则发文时说"开留言"）
- 封面：优先使用用户提供图片；未提供时按文章主题自动生成横版 16:9 封面
- 发布方式：先建草稿 → 后台预览确认 → 你说"确认发布"才正式发表
